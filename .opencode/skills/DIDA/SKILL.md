# GTD Minimal — 极简 GTD 工作流

基于滴答清单 + `dida-cli`，对话驱动。时区统一为 `Asia/Shanghai` / `+0800`。

## 清单结构

| 清单 | 用途 |
|------|------|
| 收集箱（智能清单） | 所有新任务的默认入口，随时往里扔 |
| 今天（智能清单） | 打算今天完成的任务 |
| 明天（智能清单） | 明日待办 |
| 未来七天（智能清单） | 本周后续任务 |
| **未来**（自定义清单） | 不急的、未来某天做的事 |

不使用标签。

## 工作流

### 1. 收集

将任何想法、待办快速扔进收件箱：

```
dida task create --title "..." --project <收集箱ID>
```

### 2. 早晨处理收集箱

每天早晨引导用户逐一澄清收集箱里的任务。

对每项任务：

1. **这是什么？** — 让用户简述，必要时补充到任务 `content`
2. **2 分钟能搞定吗？** — 是 → 立即去做，标记完成。否 → 继续
3. **适合今天做吗？** — 是 → 设定截止日期为今天，它会出现在「今天」清单。否 → 继续
4. **是未来某天要做的事吗？** — 是 → 移到「未来」清单。否 → 留在收集箱做参考资料，或删除
5. **确认** — 执行对应操作

### 3. 规划今日

处理完收集箱后，查看「今天」清单：

```
dida task filter --projects <今天清单概念对应ID> --status 0 --json
```

- 列出所有今日待办
- 帮用户排优先级
- 选定 1-3 个最重要任务作为今日必做

### 4. 执行

- 用户开始执行任务
- 完成后标记：`dida task complete <projectId> <taskId>`

## 快速指令

用户随时可以说：

| 用户说 | 动作 |
|--------|------|
| "记一下/收集/添加..." | 创建任务到收集箱 |
| "开始处理收件箱" | 启动收件箱逐项处理流程 |
| "今天要做什么？" | 查看今天清单，帮排优先级 |
| "完成/搞定了 X" | 标记对应任务完成 |
| "把 X 移到未来" | 将任务移到「未来」清单 |
| "复盘今天的完成情况" | 查询今日已完成任务，简要总结 |
